Tank- und Verbrauchsstatistik eines Toyota Corolla Bj. 1998 ab Juni 2014

In diesem Dokument wird die Tankstatistik eines Toyota Corolla Bj. 1998 seit 2014 fortlaufend ausgewertet. Das Auto wird durch den Urheber dieser Auswertung berwiegend innerhalb der Region Stuttgart bewegt und an stdischen (sehr selten: an der Autobahn) Tankstellen der selben Region betankt.

Dieses Dokument und der zugrundeliegende Code ist Work-In-Progress (seit 2015) und das Datum bezieht sich auf die letzte wesentliche nderung im Code. Der Datensatz wird laufend ergnzt. Die verwendete Version von R ist 4.4.3.

Bei dem Auto handelt es sich um:

  • Hersteller: Toyota
  • Modell: Corolla
  • Erstzulassung: 10.06.1998
  • Treibstoff: Benzin (E95)
  • Tankvolumen: 40 Liter
  • Leergewicht: 920kg
  • Hubraum: 1587cm3
  • Leistung: 81kw
  • Max. Umdrehungen: 6000U/min
  • Hchstgeschw.: 195km/h

Folgende Daten werden anhand der Tankquittungen und Notizen/Photos auf einem Mobiltelefon (Kilometerstnde) erfasst:

  • Datum
  • Uhrzeit
  • Getankte Liter (E95)
  • Literpreis
  • Betrag in Euro
  • Gefahrene Kilometer bis zur nchsten Betankung
  • Kilometerstand
  • Anmerkungen zu Rabatten an der Kasse, Ungenauigkeiten etc. (nicht abgebildet)

Die folgende Analyse teilt sich in folgende Abschnitte (Tabs) auf:

  • Datenaufbereitung: die Rohdaten werden eingelesen und zur weiteren Verarbeitung vorbereitet. Lcken im Datensatz werden durch Schtzung auf Grundlage benachbarter Werte und durch Rckrechnung aufgefllt. Zudem werden einige Quotienten berechnet.
  • Gesamtzeitraum: es werden kumulierte Summen, arithmetrische Mittel und Mediane (wo aussagekrftig) ber den gesamten Zeitraum berechnet und letzere beide miteinander verglichen.
  • Jhrliche Auswertung: selbiges fr die Einzeljahre.
  • Abbildungen: fr einige Variablen werden die zeitlichen Verlufe mit allen Beobachtungen abgebildet. Zudem werden die Jahressummen und -mittelwerte abgebildet. Bei letzeren werden die beiden Mittel fr jedes Jahr gegenbergestellt.
  • Data Story.
  • Alle Quelltexte, Dateien und Lizenz

Hinweise:

  • Die Datenaufbereitung und -auswertung erfolgt ausschlielich mithilfe der R-Basispakete. Dies war eine Entscheidung aus autodidaktischen Grnden und wird so beibehalten. Fr Abbildungen wird tidyverse verwendet.
  • Die Ergebnisse werden in den bersichten nicht interpretiert. Hierzu ist wurde die Data Story erstellt.
  • In diesem Markdown-Dokument ist das Skript nur Teilweise einsehbar. Vollstndige Quelltexte aller Varianten sind im Tab Alle Quelltexte und Lizenz verlinkt.

Rekonstruktion, Teil 1

Im folgenden werden Tag und Monat durch Annahme rekonstruiert und der Literpreis berechnet:

# Rekonstruktion von [1, 1]; [1, 2] (Tag und Monat)
df.raw[1, 1] <- 20; df.raw[1, 2] <- 6

# Rekonstruktion von [1, 7] (Literpreis)
df.raw[1, 7] <- df.raw[1, 8]/df.raw[1, 6]

Aufbereitung, Teil 1

Im Folgenden werden Tag, Monat, Jahr zu einer Datumsvariable zusammengefasst und die Tagedifferenz zwischen zwei Betankungen berechnet.

# Erzeugen einer Datumsvariable und Einfgen dieser 
# in die Tabelle.
Datum <- paste(df.raw$Jahr,"-",df.raw$Monat,"-",df.raw$Tag, sep="")

# Berechnen der vergangenen Tage zwischen zwei Betankungen
Tage <- as.integer(diff(df.raw$Datum))
df.raw$Tage <- c(Tage, NA)

Aufbereitung, Teil 2

Die Rekonstruktion ist an dieser Stelle abgeschlossen. Im Folgenden werden einige Hilfsobjekte aus dem Speicher entfernt, zwei Datenformate gendert und Quotienten gebildet.

## Bereinigung Arbeitsumgebung

rm(df.raw.2019, Mittel.a.2019, Datum, i, Tage)


## Bilden einiger Quotienten
# Liter je km

# Euro je km
df.raw$Euro.km <- df.raw$Euro / df.raw$km

# Kilometer je Tag
df.raw$km.Tag <- df.raw$km / df.raw$Tage

# Liter je Tag
df.raw$Liter.Tag <- df.raw$Liter / df.raw$Tage

# Euro je Tag
df.raw$Euro.Tag <- df.raw$Euro / df.raw$Tage


## Format

Aufbereitung, Teil 3

Im Folgenden werden alle Spalten gerundet.

Fr die weitere Auswertung ist dies nicht zwingend ntig, aber das Einlesen des exportierten, aufbereiteten Datensatzes in eine SQL-Datenbank wird durch eine uneinheitliche Anzahl von Nachkommastellen sehr erschwert (erzwingt unexaktes und speicherineffizentes Zahlenformat).

Um dem zu begegnen, werden alle Spalten gerundet. Einige Distanz- und Kostenspalten behalten dabei ungewohnt viele Nachkommastellen, weil grber gerundete Werte zu oft gleich zueinander sind.
